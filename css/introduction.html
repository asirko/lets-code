Le langage CSS

Avez-vous remarqué le changement ?

Effectivement, le changement d'aspect est assez violent. J'espère que vous préférez ! Je profite de vous présenter le style CSS pour améliorer légèrement l'aspect visuel de cette introduction !

J'ai, comme pour l'HTML, essayé de garder le code aussi simple que possible. Donc n'hésitez pas à jeter un oeil sur les sources de la page (et du CSS) !

Pour accéder au code CSS, vous avez 2 méthodes :

  • En ouvrant le code source de la page, vous trouverai <link rel="stylesheet" type="text/css" href="./style.css"> dans l'élément head, ./style.css est un lien vers le fichier CSS utilisé pour la page.
  • En allant directement à l'adresse du fichier CSS qui n'est autre que lets-code.fr/css/style.css.

Des langages différents mais comment les lier ?

Effectivement, les plus vifs d'entre vous ont mis le doigt ou ça fait mal ! Dans l'introduction de l'HTML, j'ai insité sur le faite qu'il y a plusieurs langages pour faire une page web. Mais ça ne vous aide pas du tout pour comprendre comment les articuler ensemble.

Alors même si je ne donnerai pas d'explications précises (on n'est que dans une simple introdution, je vous le rappelle), je peux quand même vous suggérer qu'il existe plusieurs façon de s'y prendre. On peut mélanger les deux langages et faire une belle tambouille ou essayer de séparer les deux pour rendre les choses plus claire.

Bon vous avez compris que j'ai une petite préférence pour la deuxième version ! Chacun sa place !

La question qui devrait vous turlupiner maintenant est la suivante : Si l'on sépare les deux langages (par exemple dans deux fichiers différents) comment les faire travailler ensemble ?

Là encore bougres esprits malicieux, vous êtes au coeur du problème : l'HTML est lié au contenu mais si le CSS est séparé de l'HTML, alors il le sera aussi du contenu !

L'HTML, le socle du CSS (entre autre)

L'HTML décrit qu'est-ce qui est quoi ! Donc si l'on sait identifier un élément (ou groupe d'éléments) HTML particulié(s), on n'a plus besoin du contenu ! Le contenu peut être ciblé à travers l'HTML qui le décrit.

Désolé de radoter mais c'est important !

L'HTML est l'ossature du contenu d'une page web, en suivant ses articulations, on peut atteindre les éléments souhaités. Et sans vouloir choquer vos coeurs innocents, cette ossature sert beaucoup !

Le CSS se sert ce ce squelette pour le couvrir de vêtement avec des couleurs et des formes qui accroche le regard.

Des couleurs et des formes, c'est tout ?

Oula non, pauvre fou ! Je ne veux pas me faire méprendre ! Le CSS est une discipline très vaste dont les techniques prennent énormément de temps à acquérir. Ce langage est composé de très nombreuse propriétés d'apparence (qui ne fonctionnent pas toutes très bien ensemble). Sans compter que l'apprence d'une page dépend très directement de la surface d'écran disponible. Imaginez la différence entre un smartphone et un écran plat super large connecté !

Et ces mêmes techniques ne sont que très peu de choses sans un minimum d'esprit artistique. Le CSS est pour une grande part un vecteur d'expression qui permet de donner une identité à un site internet. Il l'habile, certe, mais lui donne un corps, une identité.

De nombreux sujets sont intimement liés au design. Comme l'ergonomie qui cherche par exemple à identifier les facteurs "d'utilisabilité" d'une application pour la rendre plus simple et intuitive. Ou l'accessibilité qui assure au plus grand nombre une expérience d'utilisation optimal (en prennant en compte certains handicapes comme une diminution de l'aquité visuel ou une mauvaise perception des couleurs). D'ailleur on parle assez souvant d'UIX (pour User Interaction and Experience).

Cette page est un bien maigre exemple : elle ne prend pas en compte les écrans de petite taille, elle est très simple (pour ne pas dire plate). Mais elle permet au moins d'avoir un comptenu lisible et c'est ma principale préocupation jusqu'à la fin de cette introduction !
